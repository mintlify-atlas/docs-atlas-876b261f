---
title: "Numeric methods"
description: "Registering, reading, and storing integer and float configuration properties"
---

Figtree supports three numeric types: `int`, `int64`, and `float64`. Each type has a matching `NewXxx`, getter, and `StoreXxx` method. They are defined by the `Intable`, `Intable64`, and `Floatable` interfaces respectively, all of which are part of the `Plant` interface.

All property names are normalized to lowercase internally.

---

## Int type

### NewInt()

```go
func (p Plant) NewInt(name string, value int, usage string) Plant
```

Registers a new `int` property and wires it to the flag parser. Must be called before `Parse()`, `Load()`, or any `ParseFile()` / `LoadFile()` call.

<ParamField path="name" type="string" required>
  The flag name. Exposed as `-name=10` on the command line and as the uppercase env var `NAME`.
</ParamField>

<ParamField path="value" type="int" required>
  Default value used when the flag is not provided.
</ParamField>

<ParamField path="usage" type="string" required>
  Human-readable description displayed by `-h` and `UsageString()`.
</ParamField>

<ResponseField name="Plant" type="Plant">
  The same Plant instance, enabling method chaining.
</ResponseField>

```go
import (
    "log"
    "runtime"
    "github.com/andreimerlescu/figtree/v2"
)

figs := figtree.New()
figs.NewInt("workers", 10, "number of concurrent workers")
figs.WithValidator("workers", figtree.AssureIntInRange(1, runtime.GOMAXPROCS(0)))
if err := figs.Parse(); err != nil {
    log.Fatal(err)
}
workers := *figs.Int("workers")
```

---

### Int()

```go
func (p Plant) Int(name string) *int
```

Returns a pointer to the current `int` value for the named property. Returns `nil` if the property has not been registered. When `Options.Pollinate` is `true`, checks the environment on every call.

<ParamField path="name" type="string" required>
  The registered property name (case-insensitive).
</ParamField>

<ResponseField name="*int" type="*int">
  Pointer to the current value. Returns `nil` for unregistered properties.
</ResponseField>

```go
workers := *figs.Int("workers")
fmt.Printf("starting %d workers\n", workers)
```

---

### StoreInt()

```go
func (p Plant) StoreInt(name string, value int) Plant
```

Replaces the current value of a registered `int` property. Emits a `Mutation` on the channel when `Tracking` is enabled.

<ParamField path="name" type="string" required>
  The registered property name.
</ParamField>

<ParamField path="value" type="int" required>
  The new value.
</ParamField>

<ResponseField name="Plant" type="Plant">
  The same Plant instance, enabling method chaining.
</ResponseField>

```go
figs.StoreInt("workers", 20)
fmt.Println(*figs.Int("workers")) // 20
```

## Float64 type

### NewFloat64()

```go
func (p Plant) NewFloat64(name string, value float64, usage string) Plant
```

Registers a new `float64` property. Use for rate limits, thresholds, probabilities, or any fractional numeric configuration.

<ParamField path="name" type="string" required>
  The flag name. Exposed as `-name=0.75` on the command line.
</ParamField>

<ParamField path="value" type="float64" required>
  Default value.
</ParamField>

<ParamField path="usage" type="string" required>
  Human-readable description.
</ParamField>

<ResponseField name="Plant" type="Plant">
  The same Plant instance, enabling method chaining.
</ResponseField>

```go
figs := figtree.New()
figs.NewFloat64("threshold", 0.75, "acceptance threshold (0.0–1.0)")
figs.WithValidator("threshold", figtree.AssureFloat64InRange(0.0, 1.0))
if err := figs.Parse(); err != nil {
    log.Fatal(err)
}
threshold := *figs.Float64("threshold")
fmt.Printf("threshold: %.2f\n", threshold)
```

---

func (p Plant) Float64(name string) *float64

## Environment variable behavior

For all numeric types, the environment variable name is the uppercase form of the property name. For example, a property registered as `"workers"` is overridden by `WORKERS=20` in the environment.

When `Options.Pollinate` is `true`, the getter re-reads the environment on every call and automatically calls `StoreXxx()` if the value has changed. Without `Pollinate`, environment variables are read only once during `Parse()` or `Load()`.

```bash
WORKERS=20 THRESHOLD=0.99 go run ./myapp
```
