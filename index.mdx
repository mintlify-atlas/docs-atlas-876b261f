Figtree is a configuration management library for Go CLI applications. Import it as `figs`, configure your app's runtime properties once, and let Figtree handle flag parsing, environment variable resolution, file loading (YAML, JSON, INI), type-safe access, validation, and live mutation tracking.

## What Figtree does

Figtree wraps Go's `flag` package and adds a layered configuration system. Every property you register — a string, integer, duration, list, or map — can be populated from:

1. A config file (YAML, JSON, or INI)
2. Environment variables (uppercase key name)
3. CLI flags (`-name=value`)

Sources are applied in that order of precedence. Figtree resolves them all automatically when you call `Parse()` or `Load()`.

<Steps>
  <Step title="Install">
    Add Figtree to your Go module with a single command.

    ```bash
    go get -u github.com/andreimerlescu/figtree/v2
    ```
  </Step>
  <Step title="Define properties">
    Register every configuration property your app needs before calling `Parse()`.

    ```go
    figs := figtree.Grow()
    figs.NewString("endpoint", "http://localhost:8080", "API endpoint URL")
    figs.NewInt("workers", 4, "Number of concurrent workers")
    figs.NewBool("debug", false, "Enable debug logging")
    ```
  </Step>
  <Step title="Validate and parse">
    Attach validators, then parse flags and environment variables in one call.

    ```go
    figs.WithValidator("endpoint", figtree.AssureStringHasPrefix("http"))
    figs.WithValidator("workers", figtree.AssureIntInRange(1, 32))
    if err := figs.Parse(); err != nil {
        log.Fatal(err)
    }
    ```
  </Step>
  <Step title="Read values">
    Access any property as a typed pointer, anywhere in your program.

    ```go
    endpoint := *figs.String("endpoint")
    workers  := *figs.Int("workers")
    debug    := *figs.Bool("debug")
    ```
  </Step>
</Steps>
