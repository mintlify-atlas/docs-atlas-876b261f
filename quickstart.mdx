This guide walks you from zero to a working Go CLI application that reads configuration from flags, environment variables, and a config file — all through Figtree.

<Steps>
  <Step title="Install Figtree">
    Add the module to your project:

    ```bash
    go get -u github.com/andreimerlescu/figtree/v2
    ```

    Import it in your Go file. Aliasing to `figtree` is the convention:

    ```go
    import figtree "github.com/andreimerlescu/figtree/v2"
    ```
  </Step>

  <Step title="Create the tree">
    Choose a constructor. For most applications, `Grow()` is the right default — it enables mutation tracking so you can react to configuration changes at runtime without any extra setup.

    ```go
    figs := figtree.Grow()
    ```

    <Note>
      If you do not need to watch for changes (for example, a short-lived script), use `figtree.New()` instead. It skips the overhead of the mutations channel entirely.
    </Note>
  </Step>

  <Step title="Declare properties">
    Register every configuration property before calling `Parse()`. Each `New<Type>` method takes a name, default value, and usage string. Chain `WithValidator` calls immediately after each declaration.

    ```go
    package main

    import (
        "log"
        "time"

        figtree "github.com/andreimerlescu/figtree/v2"
    )

    func main() {
        figs := figtree.Grow()

        // String property: -endpoint "https://api.example.com"
        figs.NewString("endpoint", "http://localhost:8080", "API base URL")
        figs.WithValidator("endpoint", figtree.AssureStringHasPrefix("http"))

        // Int property: -workers 4
        figs.NewInt("workers", 4, "Number of parallel workers")
        figs.WithValidator("workers", figtree.AssureIntInRange(1, 64))

        // Bool property: -debug
        figs.NewBool("debug", false, "Enable verbose debug output")

        // UnitDuration: -timeout 30 means 30 × time.Second = 30s
        figs.NewUnitDuration("timeout", 30*time.Second, time.Second, "Request timeout")
        figs.WithValidator("timeout", figtree.AssureDurationMin(5*time.Second))
        figs.WithValidator("timeout", figtree.AssureDurationMax(5*time.Minute))

        if err := figs.Parse(); err != nil {
            log.Fatal(err)
        }

        log.Printf("endpoint : %s", *figs.String("endpoint"))
        log.Printf("workers  : %d", *figs.Int("workers"))
        log.Printf("debug    : %t", *figs.Bool("debug"))
        log.Printf("timeout  : %s", *figs.UnitDuration("timeout"))
    }
    ```

    <Tip>
      Property names are always lowercased internally. `-Endpoint`, `-endpoint`, and the environment variable `ENDPOINT` all resolve to the same key.
    </Tip>
  </Step>

  <Step title="Run it">
    ```bash
    go run main.go -endpoint https://api.example.com -workers 8
    ```

    Pass `-h` or `-help` at any time to print auto-generated usage output for every registered property.
  </Step>
</Steps>

## Load configuration from a file

Instead of (or in addition to) command-line flags, you can load configuration from a YAML, JSON, or INI file. Figtree detects the format from the file extension automatically.

<Steps>
  <Step title="Create a config file">
    <CodeGroup>
      ```yaml config.yaml
      ---
      endpoint: "https://api.example.com"
      workers: 8
      debug: true
      timeout: 30s
      ```

      ```json config.json
      {
        "endpoint": "https://api.example.com",
        "workers": 8,
        "debug": true,
        "timeout": "30s"
      }
      ```

      ```ini config.ini
      endpoint: "https://api.example.com"
      workers: 8
      debug: true
      timeout: 30s
      ```
    </CodeGroup>

    Place the file in your working directory or provide a path explicitly.
  </Step>

  <Step title="Parse from a specific file">
    Replace `figs.Parse()` with `figs.ParseFile()` and supply the path:

    ```go
    if err := figs.ParseFile("config.yaml"); err != nil {
        log.Fatal(err)
    }
    ```
  </Step>

  <Step title="Or let Figtree discover the file automatically">
    `Load()` searches the working directory for `config.yml`, `config.json`, and `config.ini` in that order. It also honours the `CONFIG_FILE` environment variable.

    ```go
    if err := figs.Load(); err != nil {
        log.Fatal(err)
    }
    ```

    You can override the default discovery path globally before creating the tree:

    ```go
    figtree.ConfigFilePath = "/etc/myapp/production.yaml"
    figs := figtree.Grow()
    // ... declare properties ...
    figs.Load()
    ```

    <Note>
      `Load()` reads the config file and environment variables. `Parse()` reads command-line flags and environment variables. Use `ParseFile()` when you want both a specific file and flags resolved together.
    </Note>
  </Step>
</Steps>

## Environment variables

Every property registered with Figtree can be set via an environment variable. The variable name is the **uppercase version of the property key**.

```bash
ENDPOINT=https://prod.example.com WORKERS=16 DEBUG=true go run main.go
```

| Source | Precedence |
|---|---|
| Environment variable | Highest — overrides flags and file values |
| Command-line flag | Overrides file values |
| Config file | Overrides compiled defaults |
| Default value | Lowest |

<Tip>
  To disable environment variable resolution entirely (useful in tests), create the tree with `figtree.With(figtree.Options{IgnoreEnvironment: true})`. This calls `os.Clearenv()` at construction time.
</Tip>

## Watch for mutations at runtime

When you use `figtree.Grow()` (or `With(Options{Tracking: true})`), the `Mutations()` channel emits a `figtree.Mutation` every time a property value changes. This is useful in long-running services that want to react to live environment variable changes without restarting.

```go
package main

import (
    "context"
    "log"
    "os"
    "os/signal"
    "syscall"
    "time"

    figtree "github.com/andreimerlescu/figtree/v2"
)

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    sigCh := make(chan os.Signal, 1)
    signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM)

    figs := figtree.Grow()
    figs.NewInt("workers", 4, "Number of parallel workers")
    figs.NewUnitDuration("interval", 5*time.Second, time.Second, "Poll interval")

    if err := figs.Parse(); err != nil {
        log.Fatal(err)
    }

    for {
        select {
        case <-ctx.Done():
            log.Println("shutting down")
            return
        case sig := <-sigCh:
            log.Printf("received signal: %v", sig)
            cancel()
        case m, ok := <-figs.Mutations():
            if !ok {
                return
            }
            log.Printf("config changed: %s  %v -> %v  (at %s)",
                m.Property, m.Old, m.New, m.When.Format(time.RFC3339))
        }
    }
}
```

Export a new value in a second terminal while the program is running:

```bash
export WORKERS=16
```

Figtree detects the change on the next getter call and sends a `Mutation` to the channel.
